# SRM Agentic AI Parking Demo Script

Use this flow for a 3-5 minute viva, project review, or recording.

## 1. Start the Runtime

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\start_project.ps1
```

macOS/Linux:

```bash
./start_project.sh
```

Open:

- Dashboard: http://127.0.0.1:8501
- API docs: http://127.0.0.1:8000/docs
- API health: http://127.0.0.1:8000/health

## 2. Show the Scenario

In the sidebar, select `Exam Rush`.

Explain that the system is simulation-first: SRM parking blocks, capacities, event pressure, entries, exits, queues, and redirects are generated by the environment while the agentic loop makes bounded decisions.

## 3. Run One Step

Click `Run One Step`.

Show:

- current decision
- source block and destination block
- vehicles redirected
- baseline comparison
- planner, critic, executor chain

## 4. Explain the Agent Loop

Open `Agent Loop`.

Walk through:

1. MonitoringAgent observes live SRM block state.
2. DemandAgent predicts pressure.
3. BayesianAgent estimates uncertainty and congestion risk.
4. PlannerAgent proposes an action.
5. CriticAgent approves, reduces, rejects, or requests replan.
6. ExecutorAgent validates executable vehicles.
7. RewardAgent scores the outcome.
8. AgentMemory stores reward and route patterns.

## 5. Show Explainability

Open `Reasoning`.

Point out:

- why the final action was selected
- what alternatives were considered
- why alternatives were rejected
- safety/risk review
- whether Gemini was requested, used, skipped, or replaced by fallback

## 6. Run Benchmark

In the sidebar:

- set benchmark episodes to `3`
- set benchmark steps to `10`
- click `Run Benchmark`

Open `Benchmark`.

Show:

- average search-time gain
- resilience gain
- hotspot reduction
- scenario-wise agentic vs baseline comparison

## 7. Export Evidence

Click `Prepare Run Report`, then download the JSON report.

This report captures:

- latest decision explanation
- KPI snapshot
- benchmark summary
- learning state
- LLM usage
- recent actions
- trace tail

## 8. Final Pitch

Use this closing line:

> This project is not only a parking simulation. It is a bounded multi-agent decision system with planner-critic-executor control, explainability, reward learning, persistent memory, baseline comparison, API access, dashboard observability, and safe LLM fallback.
